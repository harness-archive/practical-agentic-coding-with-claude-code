# 출력 인터페이스

## 1. 출력 스타일
출력 스타일은 클로드 코드가 응답을 생성할 때 따르는 지침 <br>
단순 출력 형태를 변경하는 것이 아닌 시스템 프롬프트를 직접 수정하여 응답을 변화시킨다. <br>
클로드 코드는 세가지 내장 출력 스타일을 제공한다.
1. Default
   - 소프트웨어 엔지니어링 관점에서 가장 일반적인 스타일
2. Explanatory
   - 작업을 구현하면서 결정 근거와 대안을 포함한 상세 설명
3. Learning
   - 구현은 개발자가 직접 수행하도록 안내하고 학습자를 위한 교육적 설명과 인사이트만 제공

<br>

### 1.1 출력 스타일 변경
/output-style 명령은 v2.1.73에서 더 이상 사용되지 않으며 v2.1.91에서 제거되었다. <br>
/config -> outputStyle 설정을 선택하거나 settings.json 에서 outputStyle 필드를 직접 설정한다.

<br>

### 1.2 커스텀 출력 스타일
`.claude/output-styles` 에 사용자 정의 출력 스타일 Markdown 파일을 생성하여 커스텀 출력 스타일을 지정할 수 있다. <br>
프론트매터는 메타데이터용이고, 본문이 시스템 프롬프트에 추가할 지침 <br>

커스텀 스타일은 기존 default인 소프트웨어 엔지니어링 관련 코딩 지침을 제거하기 때문에 <br> 
keep-coding-instructions:true로 설정하면 코딩 관련 기본 지침을 유지하면서 커스텀 출력 지침도 유지할 수 있다.

출력 스타일은 메인 에이전트이 시스템 프롬프트에만 영향을 미치므로 서브에이전트에 직접적인 영향을 주지 않는다.

<br>

### 1.3 출력 스타일 환각 지침
1. 출력 스타일에 URL 처리 규칙을 명시하면 WebFetch 같은 웹서치 도구 사용할 때 실제로 반환된 URL만 인용한다.
2. 불확실성 표현을 명시적으로 허용한다.
3. 출력 스타일에서 도구 결과를 기반으로 응답하도록 강제한다.

<br>


## 2. 상태 표시줄
클로드 코드 세션 터미널 하단에 표시되는 정보 영역 <br>
`/statusline` 명령어를 사용하여 설정 파일을 직접 수정할 수 있다.

### 2.1 상태 표시줄 스크립트 작동 원리
상태 표시줄 스크립트(이하 상태 스크립트)는 클로드 코드가 JSON 데이터를 stdin으로 전달받아 처리한다. <br>
스크립트는 이 정보를 가공하여 stdout으로 출력한다. <br>
