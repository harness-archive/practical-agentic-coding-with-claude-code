## 1. 출력 스타일의 개념과 내장 스타일
출력 스타일은 클로드 코드가 생성하는 응답의 형태와 톤을 결정짓는 시스템 프롬프트 가이드라인입니다. 기존 프롬프트에 내용을 덧붙이는 `CLAUDE.md`와 달리, 기본 시스템 프롬프트를 완전히 새로운 지침으로 대체하는 특징을 가집니다.

- **3가지 내장 스타일**:
  1. **Default**: 소프트웨어 엔지니어링에 최적화된 간결하고 전문적인 기본 모드입니다.
  2. **Explanatory**: 코드를 작성하며 구현 선택의 이유와 패턴을 설명해 주는 교육적인 모드입니다.
  3. **Learning**: 클로드가 직접 코드를 모두 작성하지 않고, `TODO(human)` 마커를 남겨 사용자가 직접 코딩을 실습하고 학습하도록 유도하는 협업 모드입니다.
- **적용 방법**: 대화 중 `/output-style` 명령어를 사용하거나, `.claude/settings.json` 파일의 `outputStyle` 키에 팀 표준 스타일을 지정할 수 있습니다.

## 2. 커스텀 스타일과 환각(Hallucination) 방지
내장 스타일 외에도 `.claude/output-styles/` 디렉터리에 마크다운 파일을 생성하여 응답 언어, 문서화 규칙 등을 자유롭게 정의한 커스텀 스타일을 만들 수 있습니다. 

특히 커스텀 스타일은 AI의 **환각(Hallucination) 현상을 시스템 수준에서 통제**하는 데 매우 효과적입니다.
- **URL 및 출처 검증**: "WebFetch나 WebSearch 도구를 통해 실제로 반환된 URL만 인용하고, 패턴 기반으로 추측하지 않는다"와 같은 규칙을 강제합니다.
- **불확실성 표현 허용**: "이 세부 사항에 대해 확실하지 않습니다", "추가 검증이 필요합니다"와 같이 모르는 것을 명시적으로 인정하도록 허용하여 거짓 정보 생성을 막습니다.
- **도구 결과 기반 응답**: 패키지 의존성이나 API 스펙에 대해 일반 지식이 아닌 컨텍스트 검색 도구(`Context7`) 결과를 반드시 인용하도록 요구합니다.

## 3. 상태 표시줄(Statusline) 구성 및 스크립트 최적화
상태 표시줄은 터미널 하단에 작업 디렉터리, 사용 모델, 세션 비용, 추가/삭제된 코드 라인 수, 깃(Git) 브랜치 등의 메타 정보를 실시간으로 보여주는 영역입니다.

- **작동 원리**: 클로드 코드가 최대 300ms 간격으로 세션 정보가 담긴 JSON 데이터를 `stdin`으로 전달하면, Bash, 파이썬, 또는 Node.js 스크립트가 이를 파싱하여 `stdout`으로 한 줄의 텍스트를 출력합니다.
- **시각화 및 최적화 전략**:
  - **ANSI 색상 코드**: 상태 표시줄 스크립트 내에 색상 코드(예: 노란색 모델명, 빨간색 삭제 라인 등)를 적용하여 가독성을 높일 수 있습니다.
  - **캐싱(Caching) 활용**: `git status`처럼 실행 비용이 큰 작업은 매 업데이트(300ms)마다 실행하면 성능이 저하되므로, 5초 정도의 TTL(Time-To-Live)을 가진 임시 캐시 파일을 생성하여 읽어오는 방식으로 최적화해야 합니다.

## 4. 대화형 모드 핵심 단축키
클로드 코드 세션을 키보드만으로 효율적으로 제어하기 위한 다양한 단축키가 지원됩니다.

- **Ctrl + O**: 클로드가 사용하는 도구와 결과를 더 자세히 볼 수 있는 상세 출력(Verbose) 모드를 토글합니다.
- **Ctrl + R**: 역방향 명령 기록 검색을 활성화하여 과거에 입력한 프롬프트를 빠르게 찾아 재사용합니다.
- **Esc + Esc**: 코드 수정 내용과 대화 기록을 이전 체크포인트 상태로 즉시 되감기(Rewind) 합니다.
- **Tab**: 복잡한 아키텍처 결정이나 난해한 디버깅 시 리소스를 추가로 사용하여 깊이 사고하는 **확장 사고(Extended Thinking)** 기능을 토글합니다.
- **Shift + Tab** (또는 `Alt + M`): 도구 사용 시 확인 절차를 변경하는 '권한 모드(표준/계획/자동 수락)'를 전환합니다.
- **다중 줄 및 이미지 입력**: `\` 입력 후 Enter를 누르거나 (설정 시 `Shift + Enter`), 클립보드에 있는 이미지(에러 캡처 등)를 터미널에 직접 붙여넣어 시각적 디버깅을 요청할 수 있습니다.

## 5. 빠른 명령 접두사와 백그라운드 실행
프롬프트 입력 시 특정 특수문자로 시작하면 클로드가 이를 특수 명령으로 인식하여 빠르게 처리합니다.

- **`#` (메모리 단축키)**: 입력한 내용을 프로젝트의 `CLAUDE.md` 파일에 즉시 기록하여 영구 컨텍스트로 저장합니다.
- **`!` (Blast 모드)**: `!git status`와 같이 입력하면 AI의 확인이나 해석 없이 Bash 명령어를 즉시 실행하고, 그 결과값을 대화 컨텍스트에 삽입합니다.
- **`@` (파일 멘션)**: 타이핑 시 로컬 파일 경로 자동 완성이 트리거되어 특정 파일을 컨텍스트에 쉽게 추가합니다.
- **`Ctrl + B` (백그라운드 실행)**: 빌드 도구나 테스트 러너, 개발 서버 등 장기 실행 프로세스가 작동할 때 `Ctrl + B`를 누르면, 해당 작업을 백그라운드로 전환하고 클로드와 대화를 계속 이어나갈 수 있습니다.
