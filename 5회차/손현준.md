# 클로드 코드 실전 가이드: 5장 출력 인터페이스

클로드 코드의 응답 방식과 톤을 결정하는 '출력 스타일(Output Style)' 커스터마이징, 터미널 환경을 풍성하게 해주는 '상태 표시줄(Statusline)', 그리고 작업 생산성을 극대화하는 '대화형 모드 고급 기능'을 상세한 예시 코드와 함께 정리했습니다.

---

## 1. 출력 스타일 (Output Style) 커스터마이징

출력 스타일은 기존 시스템 프롬프트에 단순히 내용을 추가하는 `CLAUDE.md`와 달리, **클로드 코드의 기본 시스템 프롬프트를 완전히 대체**하여 응답의 형태와 톤을 근본적으로 변화시키는 시스템입니다.

### 1.1 3가지 내장 스타일
*   **Default (기본)**: 소프트웨어 엔지니어링에 최적화된 모드로, 간결하고 전문적인 응답을 제공합니다.
*   **Explanatory (설명 중심)**: 코드를 작성하며 구현 선택의 이유와 패턴을 상세히 설명해 주는 교육적인 모드입니다.
*   **Learning (학습 모드)**: 클로드가 코드를 모두 작성하지 않고, 핵심 구현부에 `TODO(human)` 마커를 남겨 사용자가 직접 코딩을 실습하도록 유도하는 협업 모드입니다.

적용하려면 대화 중 `/output-style` 명령어를 사용하거나 `.claude/settings.json` 파일의 `outputStyle` 키에 지정합니다.

### 1.2 커스텀 스타일 생성과 환각(Hallucination) 방지
`.claude/output-styles/` 디렉터리에 마크다운 파일을 생성하여 응답 언어, 문서화 규칙 등을 자유롭게 정의할 수 있습니다. 특히 이 커스텀 스타일은 **AI의 환각을 시스템 수준에서 통제**하는 데 매우 효과적입니다.

다음은 환각을 방지하고 팀 표준을 강제하기 위해 작성할 수 있는 **커스텀 출력 스타일 마크다운 예시**입니다.

```markdown
# 파일 경로: .claude/output-styles/team-verified-style.md
name: team-verified-style
description: 환각 방지 지침이 포함된 팀 표준 스타일

# 팀 검증 스타일

## 핵심 원칙
모든 기술 정보는 검증 가능한 출처로 추적할 수 있어야 한다. 확실하지 않은 정보에 대해서는 "이 세부 사항에 대해 확실하지 않습니다"라고 명시적으로 인정한다.

## 검증 요구 사항
- 도구 검증 없이 URL 제공 금지 (반드시 WebFetch나 WebSearch 결과만 인용)
- 파일 시스템 확인 없이 패키지 버전 번호 제공 금지
- 일반적인 지식(Pre-trained knowledge)에 의존한 API 동작 주장 금지 (Context7 도구 사용)

## 응답 형식 및 신뢰도 마커
모든 기술적 주장에 출처 표시와 신뢰도를 포함한다:
- 파일 기반: (출처: 경로/파일명: 줄번호)
- 웹 기반: (출처: WebFetch/WebSearch를 통한 URL)
- 신뢰도: [검증됨], [가능성 높음], [불확실] 중 하나를 반드시 표기
```

## 2. 상태 표시줄 (Statusline) 구성 및 성능 최적화

상태 표시줄은 터미널 하단에 작업 디렉터리, 사용 모델, 세션 비용, 추가/삭제된 코드 라인 수, 깃(Git) 브랜치 등을 실시간으로 시각화해 주는 기능입니다.

### 2.1 작동 원리 및 기본 스크립트
클로드 코드는 최대 300ms 간격으로 세션 정보가 담긴 JSON 데이터를 `stdin`을 통해 스크립트로 전달합니다. 스크립트는 이를 파싱하여 `stdout`으로 한 줄의 텍스트를 출력합니다.

다음은 `jq` 도구를 활용하여 **가장 기본적으로 모델 이름과 디렉터리를 표시하는 Bash 스크립트 예시**입니다.

```bash
# 파일 경로: ~/.claude/statusline.sh
#!/bin/bash

# stdin에서 JSON 데이터 읽기
input=$(cat)

# 모델 표시 이름과 현재 디렉터리 추출
MODEL_DISPLAY=$(echo "$input" | jq -r '.model.display_name')
CURRENT_DIR=$(echo "$input" | jq -r '.workspace.current_dir')

# 포매팅 된 상태 표시줄 출력 (예: [Opus] my-project)
echo "[$MODEL_DISPLAY] ${CURRENT_DIR##*/}"
```

### 2.2 성능 최적화 (캐싱 적용)
상태 표시줄은 300ms마다 업데이트되므로 `git branch` 확인처럼 무거운 프로세스를 매번 실행하면 심각한 성능 저하가 발생합니다. 따라서 **임시 캐시 파일을 활용한 TTL(Time-To-Live) 최적화**가 필수적입니다.

다음은 **5초 캐싱을 적용하여 Git 브랜치를 표시하는 최적화 스크립트 예시**입니다.

```bash
#!/bin/bash
input=$(cat)

CACHE_FILE="/tmp/claude-statusline-git-cache"
CACHE_TTL=5 # 5초 캐시 유효 시간

# 캐시 확인: 유효 시간 내에 있는지 체크
if [ -f "$CACHE_FILE" ]; then
  CACHE_AGE=$(($(date +%s) - $(stat -c%Y "$CACHE_FILE" 2>/dev/null)))
  if [ "$CACHE_AGE" -lt "$CACHE_TTL" ]; then
    GIT_BRANCH=$(cat "$CACHE_FILE")
  fi
fi

# 캐시가 없거나 만료되었으면 git 명령어로 새로 가져오기
if [ -z "$GIT_BRANCH" ]; then
  if git rev-parse --git-dir > /dev/null 2>&1; then
    GIT_BRANCH=$(git branch --show-current 2>/dev/null)
    echo "$GIT_BRANCH" > "$CACHE_FILE"
  fi
fi

MODEL_DISPLAY=$(echo "$input" | jq -r '.model.display_name')
CURRENT_DIR=$(echo "$input" | jq -r '.workspace.current_dir')

# ANSI 색상 코드를 섞어 출력 (예시)
YELLOW='\033[0;33m'
CYAN='\033[0;36m'
NC='\033[0m'

echo -e "${YELLOW}[$MODEL_DISPLAY]${NC} ${CURRENT_DIR##*/} ${CYAN}$GIT_BRANCH${NC}"
```

## 3. 대화형 모드 핵심 제어 및 단축키

키보드만으로 세션을 완벽히 통제하고 작업 효율을 높이는 단축키와 입력 패턴입니다.

### 3.1 생산성을 높이는 핵심 단축키
*   **`Ctrl + O` (상세 출력 모드)**: 클로드가 내부적으로 어떤 도구를 사용하고 어떤 결과를 받았는지 상세한 과정(Verbose)을 토글합니다.
*   **`Ctrl + R` (명령 기록 검색)**: 과거에 입력한 프롬프트 히스토리를 역방향으로 탐색하여 빠르게 재사용합니다.
*   **`Esc + Esc` (즉시 되감기)**: 잘못된 수정이나 의도치 않은 대화가 발생했을 때 이전 체크포인트 상태로 코드를 즉시 롤백(Rewind) 합니다.
*   **`Tab`**: 복잡한 추론이나 아키텍처 결정이 필요할 때 토큰 예산을 추가로 사용하는 **확장 사고(Extended Thinking)** 모드를 켜거나 끕니다.
*   **`Shift + Tab` (또는 `Alt + M`)**: 도구 사용 승인 방식인 **권한 모드(표준/계획/자동 수락)**를 빠르게 전환합니다.
*   **`Ctrl + B` (백그라운드 실행)**: 빌드나 테스트 서버 같은 장기 실행 프로세스가 작동할 때 누르면, 해당 작업을 백그라운드로 밀어 넣고 클로드와 대화를 계속할 수 있습니다.

### 3.2 입력 편의 기능과 빠른 명령 접두사
*   **다중 줄 입력**: 백슬래시(`\`)를 입력 후 Enter를 누르거나, 에디터 세팅에 따라 `Shift + Enter`로 줄 바꿈을 할 수 있습니다. 
*   **이미지 기반 디버깅**: 클립보드에 있는 에러 스크린샷이나 UI 시안을 터미널에 직접 붙여넣어 시각적 분석을 요청할 수 있습니다.
*   **`#` (메모리 단축키)**: 입력한 내용을 프로젝트의 `CLAUDE.md` 파일에 즉시 기록하여 영구 컨텍스트로 저장합니다.
*   **`!` (Blast 모드)**: `!git status`와 같이 입력하면 AI의 확인이나 해석 없이 즉시 터미널 셸 명령을 실행하고, 그 결과값을 대화 컨텍스트에 삽입합니다.
*   **`@` (파일 멘션)**: 타이핑 시 로컬 파일 경로 자동 완성이 트리거되어 분석할 파일을 컨텍스트에 직관적으로 추가합니다.
