# Chapter 05 출력 인터페이스

## 5-1 출력 스타일 커스터마이징
### 1) 출력 스타일의 핵심 개념
- 클로드 코드의 시스템 프롬프트를 직접 수정하여 응답의 근본적인 특성을 변화시킨다. 
- 출력 스타일을 적절히 설정하면 로컬 스크립트 실행이나 파일 작업 같은 핵심 기능은 유지하면서도 응답의 성격을 완전히 다르게 조정할 수 있다. 

### 2) 내장 출력 스타일 살펴보기
- 1) Default 스타일 : 개발작업에 적합하며 코드 분석과 생성, 디버깅 등의 작업에서 전문적인 응답 제공
- 2) Explanatory 스타일 : 작업을 완료하는 동안 구현 선택에 대한 교육적 인사이트를 함께 제공한다. 코드를 작성하거나 수정할 때 왜 그런 결정을 내렸는지 설명하므로 코드 리뷰 학습이나 새로운 기술 스택을 익힐 때 쓸 수 있다. 
- 3) Learning 스타일 : 클로드가 인사이트를 공유하고 전략적 코드 조각만 기여하는 협업 모드다. 

### 3) 출력 스타일 변경 방법
- /output-style 명령을 사용, 인자없이 실행하면 사용가능한 스타일 목록이 표시되고 원하는 스타일을 선택할 수 있다. 
- 스타일 변경은 프로젝트 수준의 로컬 설정 파일인 .claude/settings.local.json에 저장

### 4) 커스텀 출력 스타일 생성하기 
- 사용자 수준 스타일 경로 : ~/.claude/output-styles/
- 프로젝트 수준 스타일 경로 : .claude/output-styles/

### 5) 팀 표준 출력 스타일 구축
- 표준 스타일을 설계할 때는 다음 요소를 고려해야 한다. 
- 1) 팀의 코딩 컨벤션과 문서화 표준을 스타일에 반영한다. 
- 2) 응답의 상세 수준을 정의한다. 
- 3) 보안 관련 지침을 포함한다. 

## 5-2 상태 표시줄
### 1) 상태 표시줄 설정 방법
- /statusline 명령을 사용하거나 설정 파일을 직접 편집해서 설정할 수 있다. 
- 해당 명령을 사용하면 클로드 코드가 설정을 안내하며 현재 터미널 프롬프트와 유사한 형태로 상태 표시줄을 구성해 준다. 
- 추가 지침도 함께 전달이 가능하다. 
- 설정 파일을 직접 편집하려면 statusLine 키를 .claude/settings.json 파일 또는 ~/.claude/settings.json 파일에 추가한다. 
    ```json
    {
      "statusLine": {
        "type": "command",
        "command": "~/.claude/statusline.sh",
        "padding": 0
      }
    }
    ```
    - type : command로 설정하여 외부 스크립트를 실행하도록 지정
    - command : 상태 표시줄 텍스트를 생성할 스크립트의 경로를 지정
    - padding : 상태 표시줄 주변의 여백을 조절

### 2) 문제 해결
- 상태 표시줄이 예상대로 표시되지 않을 때 확인해야할 사항이 있다. 
- 먼저 스크립트 실행 권한이 있는지 확인
    ```bash
    $ chmod +x ~/.claude/statusline.sh
    ```
- 다음으로 stdout 으로 출력하는지 확인
- stderr로 출력하면 상태 표시줄에 나타나지 않는다. 

## 5-3 대화형 모드 활용
- 대화형 모드의 키보드 단축키와 입력기능은 클로드 코드를 효율적으로 조작하는 방법을 제공한다. 
- 1) Ctrl + C : 현재 입력이나 생성을 취소하는 표준 인터럽트 신호
- 2) Ctrl + D : 클로드 코드 세션 자체를 종료하는 EOF 신호
- 3) Ctrl + L : 터미널 화면을 지유지만 대화기록은 유지한다. 
- 4) Ctrl + O : 상세 출력 모드를 토글 한다. 이 모드에서는 클로드가 사용하는 도구와 그 결과가 더 자세히 표시된다. 디버깅이나 클로드의 동작 방식을 이해할 때 유용하다. 

## 1) 고급 제어 기능
- 1) 화살표 키 : 이전 입력 기록을 탐색 할수 있다.
- 2) Esc : 두 번 연속으로 누르면 코드와 대화를 이전 상태로 되감을 수 있다. 클로드가 수행한 변경이 기대와 다르거나 실수가 있을 때 이 단축키로 빠르게 이전 상태로 복원이 가능하다. 
- 3) Tab : 확장 사고 기능을 토글한다. 확장 사고가 활성화 되면 클로드가 복잡한 문제애 대해 더 깊이 생각하고 체계적인 답변을 제공한다. 
- 4) Shift + Tab : 권한 모드를 전환한다. 
- 5) 빠른 명령 접두사 
  - #은 메모리 단축키, #으로 시작하는 입력은 프로젝트의 CLAUDE.md 파일에 추가된다. 
  - /은 내장 명령과 커스텀 명령, 스킬 등을 호출
  - !은 bash모드를 활성화 
  - @은 경로 멘션을 트리거 
- 6) Ctrl + R : 명령 기록 검색을 활성화 
  - 이전에 입력한 프롬프트를 키워드로 검색할 수 있어 비슷한 작업을 반복할 때 사용
