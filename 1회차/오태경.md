# Chapter 01. 클로드 코드 시작하기

## 클로드 vs 클로드 코드 vs 클로드 코드 CLI
-   클로드 : 앤트로픽이 만든 AI 모델로써 Opus, Sonnet, Haiku가 있다.
-   클로드 코드 : 앤트로픽이 SW 개발에 특화해 만든 에이전틱 코딩 도구, 개발 워크 플로우 전반을 자동 수행
-   클로드 코드 CLI : 클로드 코드를 터미널에서 실행할 수 있는 Command Line Interface

<br>

## 앤트로픽과 에이전틱 AI의 부상
LLM 개발사마다 시장에 접근하는 방식이 다르다.

**OpenAI**
-   텍스트, 영상 이미지 생성에 강점

**구글** **(제미나이)**
-   검색과 클라우드

**앤트로픽**
-   에이전틱 AI의 제품화에 집중

<br>

### 에이전틱 AI란?
**에이전틱 AI**
-   사용자의 질문에 답변하는 수준을 벗어나 스스로 문제를 분석, 도구를 활용하여 자율적으로 수행하는 AI
-   이러한 방식의 핵심에는 **MCP**가 자리잡고 있다.

**MCP**
-   Model Context Protocol
-   AI 모델이 외부 데이터 소스 및 도구와 상호작용할 수 있도록 표준화된 프로토콜

<br>

### 에이전틱 코딩이란
AI가 파일 시스템 탐색, 코드 수정, 테스트까지 자율로 수행하는 방식

<u>개발자가 높은 수준의 목표를 제시하면 AI는 하위 작업으로 분해-실행 계획 수립-완수한다.</u>

이 과정을 **탐색-계획-구현-커밋 워크플로**라고 한다.


### 에이전틱 루프 : 클로드 코드의 동작 원리
- <u>클로드 코드는 에이전틱 루프라는 순환 구조로 작동한다.</u>

1.   **컨텍스트 수집**
    -   프로젝트의 파일 구조 탐색후 코드 수집 및 정보 수집하여 메모리 로딩
    -   CLAUDE.md에 기록된 프로젝트 규칙, 코드 스타일, 테스트 절차 등도 컨텍스트에 해당한다.
2.   **작업 수행**
    -   컨텍스트 확보 후 실행 계획을 세우고 도구를 호출한다.
    -   클로드 코드의 **내장 도구**는 크게 파일 조작, 검색, 셀 실행, 웹 접근 4가지로 나뉜다.
    -   그 외에 서브에이전트를 생성하는 Agent, 작업 목록 관리하는 TodoWrite 등 **보조 도구**도 존재한다.
3.   **결과 확인**
    -   클로드 코드 자신이 수행한 작업을 스스로 검증하는 단계
    -   코드를 작성했다면 테스트를 실행하고 빌드가 정상적으로 되는지 검사한다.
    -   문제가 발견되면 클로드 크도는 컨텍스트 수집 단계로 돌아가 수정을 반복한다.

-   위와 같은 **에이전틱 루프는 서브에이전트를 통해 확장될 수 있다.**
-   메인 에이전트가 도구로 서브에이전트를 생성하면, 서브에이전트는 독립된 컨텍스트에서 탐색을 수행하고 결과를 요약하여 메인 에이전트에 반환한다.


<br>

### 터미널에서 시작하는 AI 코딩
-   클로드 코드는 다른 AI 플랫폼과 차이는 CLI이다. 유닉스 철학인 모듈성을 따르기에 기존 유닉스 도구와 결합해 사용 가능

    -   ```shell
        $ cat securiy_logs_*.log | cluade -p "이 로그들을 분석해서 해킹 시도 패턴을 찾아서 위협 수준과 함께 보고해줘"
        ```

    -   '보안 로그 파일의 내용을 모두 읽어라' 명령어의 출력을 다른 명령어인 프롬프트 입력으로 전달한다.

<br>

## 컨텍스트 윈도우
### 컨텍스트 윈도우와 20만 토큰 컨텍스트의 의미

-   **컨텍스트 윈도우**
    -   클로드가 한번의 추론에서 사용할 수 있는 토큰의 총량을 말한다.
    -   컨텍스트 윈도우에는 여러 정보가 순서대로 적재된다.
    -   가장 먼저 CLUADE.MD 기반의 메모리가 로드된다.
        -   엔터프라이즈 정책
        -   프로젝트 (./CLAUDE.md)
        -   사용자 메모리 (~/.claude/CLAUDE.md)
    -   프롬프트, 클로드 코드의 응답, 도구 호출과 결과가 누적되면서 컨텍스트가 쌓인다.
-   클로드 코드가 지원하는 컨텍스트 윈도우는 **20만 토큰**이다.
    -   20만 토큰을 모두 채우는것은 좋지 않다.
    -   토큰수가 증가하면 정확도가 떨어지고 추론 능력이 약화된다.
    -   컨텍스트는 한정된 자원이기에 상황에 맞는 전략으로 토큰 사용을 최적화해야한다.

<br>

### 컨텍스트 윈도우 관리 전략
클로드 코드의 컨텍스트는 크게 다음 다섯가지 전략을 기조로 관리한다.
1.   **컴팩션**
    -   컨텍스트 사용량이 98% 도달하면 **스스로 이전 대화를 자동 요약**한다.

2.   **/clear 명령**
    -   대화 기록을 완전히 초기화한다.
3.   **점진적 공개**
    -   **클로드 코드** **스킬 시스템이 이 원칙을 따른다.**
    -   모든 정보를 처음부터 로드하는 대신 가벼운 식별자만 유지하다가 도구를 통해 데이터를 동적 로드한다.
4.   **서브에이전트 활용**
    -   서브 에이전트는 메인 에이전트와 별도로 독립된 20만 토큰의 컨텍스트 윈도우를 사용이 가능하고 병렬 실행도 가능하다.
5.   **구조화된 메모 작성**
    -   에이전트가 컨텍스트 윈도우 외부에 마크다운 파일로 체크리스트를 작성하고 필요할 때 다시 불러오는 방식

     
### 다섯 가지 접근 영역과 핵심 기능
클로드 코드의 접근 영역

1.   파일 시스템
2.   터미널
3.   웹
4.   MCP 서버
5.   에디터 컨텍스트 :  VS CODE, JetBrain 등 IDE 확장


클로드 코드의 4대 핵심 기능

1.   자연어 기반의 기능 구축
2.   심층 디버깅 및 문제 해결
3.   코드베이스 탐색과 분석
4.   반복 태스크 자동화
    -   .claude/commands 디렉터리에 프롬프트 템플릿을 저장하면 팀원이 공통 워크플로를 슬래시 명령으로 호출할 수 있다.
    -   claude.md에 프로젝트별 코드 스타일 가이드라인, 테스트 절차를 기록하면 해당 규칙을 자동 참조한다.

## 클로드 코드 기초     
### 대화형 모드와 비대화형 모드

클로드 코드 CLI의 명령어는 파이프 (|) 로 다른 명령과 결합이 가능하여 강력한 자동화 워크플로 형성 가능.

대화형 모드는 플래그 없이 claude 명령만으로 실행하는 대화형 세션, claude명령으로 세션이 시작되면 프로젝트 구조를 자동으로 인식한다.

```
project-root/
├── .claude/
│   ├── commands/        # 커스텀 슬래시 명령어
│   ├── agents/          # 커스텀 에이전트 정의
│   ├── skills/          # 에이전트 스킬 모듈
│   ├── hooks/           # 이벤트 훅 스크립트
│   ├── output-styles/   # 출력 스타일 템플릿
│   └── settings.json    # 프로젝트별 클로드 코드 설정
└── CLAUDE.md            # 프로젝트 지침서(컨텍스트로 자동 로드)
```

세션은 exit 명령이나 ctrl+c로 종료할 수 있고 자동으로 저장된다.

## 체크포인트

체크포인트

-   파일 편집을 자동으로 추적하고, 사용자가 원치 않는 변경을 이전 상태로 되돌려 실행 취소하는 세션 수준의 안전망
-   사용자가 새로운 프롬프트를 입력할때마다 새 체크포인트를 자동으로 생성한다.
-   체크포인트는 세션 전체에 걸쳐 지속되며 저장기간은 30일이다.

변경 사항 되감기

-   /rewind or ESC 두번누르면 체크포인트 목록중 원하는 지점으로 선택가능

복구옵션 3가지

1.   대화만 : 클로드 코드 응답이 마음에 들지 않아 다른 방향으로 다시 시도하고싶지만 이미 작성된 코드는 유지하고 싶을 때
2.   코드만 : 클로드 코드가 잘못된 방향으로 수정했을 때, 대화의 맥락은 보존하면서 파일 변경 사항만 복원
3.   코드와 대화 모두 : 작업 전체를 특정 시점으로 되돌리고 처음부터 다시 시작하고 싶을 때
