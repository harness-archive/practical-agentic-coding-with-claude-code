# 클로드 코드 시작하기

## 1-1 클로드 코드와 에이전틱 코딩
### 1) 클로드 
  - 앤트로픽이 만든 AI 모델
### 2) 클로드 코드
- 앤트로픽이 소프트웨어 개발에 특화해 만든 에이전틱 코딩 도구
- 코드 베이스를 읽고 파일 편집, 명령어 실행 . 개발 워크플로를 전반을 자율적으로 실행
### 3) 클로드 코드 CLI
- 클로드 코드를 터미널에서 실행하는 인터페이스를 가리키는 말
### 4) 클로드 코드 동작 원리
- **에이전틱 루프**라는 순환구조로 작동
- 컨텍스트를 수집하고 작업을 수행하고 결과를 확인한다. 문제가 있으면 다시 처음으로 돌아간다.

1단계 : 컨텍스트 수집
- 프롬프트를 받으면 즉시 코드 실행을 하지 않고 파일 구조 탐색 후 관련 코드를 검색하여 필요한 정보를 수집

2단계 : 작업 수행
- 컨텍스트를 확보하면 실행계획을 세우고 도구를 호출한다.
- 클로드 코드의 내장 도구는 파일 조작, 검색, 셀실행, 웹접근 이렇게 4가지로 구성
- 상황에 따라 도구를 선택하고 필요하면 같은 단계 안에서 여러 도구를 반복 호출ㄷㄷㄷ

3단계 : 결과 확인
- 자신이 수행한 작업을 스스로 검증한다.
- 테스트를 실행해 통과 여부를 확인하고 린터로 코드 품질을 점검하며 빌드가 정상완료 되는지 검사한다. 
- 문제가 발견되면 클로드 코드는 컨텍스트 수집 단계로 돌아가 수정을 반복한다

즉, "사용자 등록 폼에 입력 검증을 추가해줘" 라고 입력하면 클로드 코드는 관련 파일을 탐색 검증 로직 설계, 테스트 실행, 오류 수정까지 에이전틱 루프를 순환하며 자율 수행한다.

### 5) 컨텍스트 윈도우와 20만 토큰 컨텍스트 의미 
**컨텍스트 윈도우(Context Windoww)**
- 클로드가 한 번의 추론에서 참조할 수 있는 토큰의 총량이다.
- 가장 먼저 CLAUDE.md 기반의 메모리가 로드 된다. 
- 엔터프라이즈 정책, 프로젝트 메모리(./CLAUDE.md), 사용자 메모리(~/.claude/CLAUDE.md), 프로젝트별 로컬 메모리가 순서대로 적재되며 현재 작업 디렉토리에서 루트 디렉토리까지 재귀적으로 탐색된다.
- 토큰 수가 증가하면 정보 회상의 정확도가 떨어지고 장거리 추론 능력이 약화된다. 

### 6) 컨텍스트 윈도우 관리 전략
첫번째 : 컨텍스트 사용량이 한계의 98% 도달시 클로드 코드는 이전 대화를 자동 요약
- 이를 통해 긴 작업에서도 에이전트가 중단되지 않는다

두번째 : /clear 명령
- 완전 초기화 시켜 새작업으로 시작

세번째 : 점진적 공개 
- 모든 정보를 처음부터 로드하는 대신 가벼운 식별자(파일경로, 링크)만 유지하다가 필요할 때 도구로 데이터를 동적로드

네번째 : 서브에이전트 활용
- 각 서브에이전트는 메인 에이전트와 별도로 20만 토큰의 독립된 컨텍스트 윈도우를 사용한다. 

다섯번째 : 구조화된 메모 작성
- 에이전트가 컨텍스트 윈도우 외부에 마크다운 파일로 체크리스트를 작성하고 필요할 때 다시 불러오는 방식
- 대규모 마이그레이션이나 다단계 수정 작업에서 진행 상황을 추적하는데 적합

### 7) 클로드 코드의 4대 핵심 기능
첫번째 : 자연어 기반의 기능 구축
- 개발자가 자연어로 구축하고자 하는 기능을 설명하면 클로드 코드가 스스로 계획을 세우고 코드를 작성한 뒤 작동 여부까지 검증
- 개발자는 코드 구현보다 상위 수준의 요구사항과 설계에 집중할 수 있다. 
두번째 : 심층 디버깅 및 문제 해결
- 버그를 설명하거나 오류 메시지를 입력하면 클로드코드가 코드베이스 전체를 분석하여 원인을 식별하고 수정을 제안한다.
세번째 : 코드베이스 탐색과 분석
- 프로젝트의 구조나 특정로직에 대해 질문하면 코드 전반을 탐색해 분석 답변을 제공
네번째 : 반복 태스크 자동화
- 린트 문제해결, 깃 병합 충돌 해결, 릴리스 노트 작성등 번거로운 작업을 자동화 한다. 

### 8) 엔터프라이즈 환경에서의 클로드 코드
- 클로드코드는 엔터프라이즈 환경의 보안 요구사항을 충족하도록 설계되어있음
- 팀이나 엔터프라이즈 요금제의 경우 중앙 집중식 과금과 팀관리를 제공하고 베드락, 버텍스 AI, 파운드리를 사용하면 각 클라우드 플랫폼의 자체 인증 체계를 따른다.

## 1-3 클로드 코드 CLI 기초
- 공식 문서 통해서 더 많은 내용 숙지 필요

### 1) 대화형 모드와 비대화형 모드 
- 대화형 모드 
  - claude 명령만으로 실행하는 REPL 스타일의 대화형 세션
  - 세션이 시작되면 클로드는 프로젝트 구조를 자동으로 인식
  - 프로젝트 루트의 CLAUDE.md를 자동으로 읽어 컨텍스트로 활용

- 필수 슬래시 명령어 
  - /help : 사용 가능한 모든 명령어와 단축키 확인
  - /clear : 현재 대화 컨텍스트 초기화
  - /rewind : 이전상태 복구
  - /exit : 세션 종료 명령어 

- 비대화형 모드 
  - 헤드리스 모드라고 하며 -p 또는 --print 플래그를 사용
  ```
  $ claude -p "이 함수의 복잡도를 분석해 줘"
  ```
  
### 2) 시스템 상태 진단과 문제 해결
- claude doctor 명령을 사용하면 설치가 제대로 됐는지를 확인하고 실행중에 발생하는 환경문제를 파악할 수 있다. 

### 3) 체크포인트와 되감기 시스템
- 자율적인 코드 수정 덕에 클로드 코드가 강력한 생산성 도구로 인정받지만 동시에 원치않은 변경이 일어날수 있다. 
- 클로드코드에서는 **체크포인팅** 이라는 자동 안정망 시스템이 있어 파일 편집을 자동으로 추적하고 사용자가 원치 않는 변경 사항을 이전 상태로 되돌려 빠르게 실행 취소 할수 있게 하는 세션 수준 안전망을 제공

## 1-4 프로젝트로 특징 이해하기 
### 1) 프로젝트 컨텍스트의 이해
- 클로드 코드는 claude 명령이 실행되는 디렉터리를 기준으로 프로젝트의 전체 구조를 파악
- 세션이 시작되면 다음과 괕은 질문으로 프로젝트 구조를 파악
```aiignore
> 이 프로젝트에 대해서 분석해 줘.
> 이 프로젝트의 기술 스택은 어떻게 구성이 되어있는지 알려줘.
> 이 프로젝트의 전체 디렉터리/파일 구조에 대해서 상세하게 분석해서 보고서 형식으로 마크다운 문서를 생성해줘.
```
